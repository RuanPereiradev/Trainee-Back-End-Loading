Visão Geral

TeamFlow é um sistema completo para gerenciamento de equipes, setores e projetos com controle de acesso baseado em papéis (Diretor, Coordenador, Membro). Esta API RESTful fornece todas as operações necessárias para gestão organizacional. O projeto usa as seguintes tecnologias: . Fastify . TypeScript . Prisma . PostgreSQL . Docker/Docker Compose . JWT . Bcrypt . Jest . Swagger (OpenAPI) . Insomnia/Postman . Git/GitHub . ESLint/Prettier . Zod
Além disso, utilizei como arquitetura de desenvolvimento o Clean Archtecture, adotando padrões de resposta para api: Result Patern, além de classe de auditoria que pode ser vista em AuditableEntity

Autenticação

Todas as rotas (exceto /login/auth) requerem autenticação via Bearer Token JWT.

Estrutura do Token JWT

O token contém as informações do usuário e sua role:

{
  "userId": "string",
  "role": "DIRETOR|COORDENADOR|MEMBRO",
  "iat": number,
  "exp": number
}

Uso em requisições: Adicione o cabeçalho Authorization: Bearer <seu-token-jwt> em todas as requisições autenticadas.

Modelos de Dados

Estruturas de dados principais utilizadas na API TeamFlow.

Usuário (User)

{
  id: string,
  name: string,
  email: string,
  password: string, // hashed
  role: "DIRETOR" | "COORDENADOR" | "MEMBRO",
  createdAt: Date,
  updatedAt: Date,
  deletedAt: Date | null
}

Setor (Sector)

{
  id: number,
  name: string,
  description: string,
  createdAt: Date,
  updatedAt: Date,
  deletedAt: Date | null
}

Projeto (Project)

{
  id: string,
  name: string,
  description: string,
  goals: string,
  status: "PLANEJADO" | "EM_ANDAMENTO" | "PAUSADO" | "CONCLUIDO",
  sectorId: number,
  userId: string, // criador
  createdAt: Date,
  updatedAt: Date,
  deletedAt: Date | null
}

Membership (Vínculo)

{
  id: string,
  userId: string,
  projectId: string,
  joinedAt: Date,
  leftAt: Date | null,
  createdAt: Date,
  updatedAt: Date
}

Permissões por Papel

Definição de permissões para cada tipo de usuário no sistema.

Diretor

  • Acesso completo ao sistema
  • Gerenciar usuários, setores, projetos
  • Definir papéis de usuários
  • Adicionar/remover membros de qualquer projeto
  • Restaurar recursos desativados

Coordenador

  • Gerenciar projetos dos quais participa
  • Adicionar/remover membros (exceto Diretores)
  • Editar dados dos projetos que coordena
  • Não pode alterar papéis de usuários
  • Não pode criar setores

Membro

  • Visualizar projetos dos quais participa
  • Atualizar próprio perfil
  • Sair de projetos
  • Não pode gerenciar estrutura organizacional
  • Não pode adicionar/remover outros membros

Endpoints Detalhados

Principais endpoints com exemplos de requisição e resposta.
POST /login/auth
Público

Autenticar usuário e obter token JWT

Body da requisição:

{
  "email": "usuario@empresa.com",
  "password": "senha123"
}

Resposta (200):

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "uuid",
    "name": "João Silva",
    "email": "joao@empresa.com",
    "role": "DIRETOR"
  }
}

Resposta (401 - Credenciais inválidas):

{
  "message": "Credenciais inválidas"
}
GET /users/pagination
Diretor

Listar usuários com paginação

Query Parameters:

?page=0&pageSize=10&role=DIRETOR&search=joao

Resposta:

{
  "data": [...],
  "pagination": {
    "page": 0,
    "pageSize": 10,
    "total": 45,
    "totalPages": 5
  }
}
POST /memberships/join
Diretor

Adicionar usuário a um projeto

Body da requisição:

{
  "userId": "user-uuid",
  "projectId": "project-uuid"
}

Regras de Negócio:

  • Um projeto só pode ter 1 Diretor
  • Um projeto só pode ter 1 Coordenador
  • Usuário não pode estar duplicado no mesmo projeto

Exemplos de Uso

1. Fluxo de Autenticação

# 1. Login
curl -X POST http://localhost:3000/login/auth \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@empresa.com","password":"Senha@123"}'

# 2. Usar token em requisições subsequentes
curl -X GET http://localhost:3000/users \
  -H "Authorization: Bearer <token-jwt>"

2. Criar Estrutura Organizacional

# 1. Criar setor (como Diretor)
curl -X POST http://localhost:3000/api/sectors \
  -H "Authorization: Bearer <token>" \
  -d '{"name":"TI","description":"Tecnologia da Informação"}'

# 2. Criar projeto no setor
curl -X POST http://localhost:3000/api/projects \
  -H "Authorization: Bearer <token>" \
  -d '{"name":"Sistema Interno","sectorId":1,"status":"PLANEJADO"}'

# 3. Adicionar membros ao projeto
curl -X POST http://localhost:3000/api/memberships/join \
  -H "Authorization: Bearer <token>" \
  -d '{"userId":"uuid-usuario","projectId":"uuid-projeto"}'

Troubleshooting

Problemas Comuns

Problema Solução
Erro de conexão com banco de dados Verificar DATABASE_URL no .env. Confirmar se PostgreSQL está rodando.
Erro "Token inválido" Verificar se token não expirou. Confirmar se JWT_SECRET está configurado.
Erro de permissão Verificar role do usuário no token. Confirmar se usuário tem acesso ao recurso.
Erro de validação Verificar schema Zod nas rotas. Confirmar tipos e formatos dos dados.
